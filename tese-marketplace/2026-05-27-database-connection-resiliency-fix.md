# Database Connection Resiliency and Deployment Engineering Fix`n`n## Metadata`n- **Date:** 2026-05-27`n- **Project:** Tese Marketplace`n- **Engineer:** Gemini CLI`n- **Status:** Resolved`n`n## Issue Summary`nProduction microservices (`auth-api`, `catalog-api`, `order-api`, `store-api`, `brain-api`, `chat-api`) were stuck in a restart loop on the VPS. The primary failure mode was a `psycopg2.OperationalError` citing host resolution failures or password authentication errors.`n`n## Root Cause Analysis`n1. **Unencoded Credentials:** The database password contained an `@` symbol (`tese@1234`). When manually concatenated into a `DATABASE_URL` string, it was not URL-encoded. This caused the URL parser to misinterpret `1234@db` as the host, which is invalid.`n2. **Environment Inconsistency:** Different layers (Docker Compose, Shell, Pydantic) handled the special characters differently, leading to a state where the database user password and the application config were perpetually out of sync.`n3. **Line Ending Mismatch:** The `entrypoint.sh` scripts had Windows (CRLF) line endings. In the Alpine-based Docker images, this resulted in `exec ./entrypoint.sh: no such file or directory` errors.`n`n## Engineering Resolution`n### 1. Type-Safe Configuration`nRefactored all microservice `config.py` files to accept granular variables instead of a raw URL string:`n- `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`, `DB_NAME``n`nImplemented programmatic URL construction using SQLAlchemy:`n```python`nfrom sqlalchemy.engine import URL`n`n@property`ndef DATABASE_URL(self) -> str:`n    return str(URL.create(`n        drivername=\"postgresql+psycopg2\",`n        username=self.DB_USER,`n        password=self.DB_PASSWORD,`n        host=self.DB_HOST,`n        port=self.DB_PORT,`n        database=self.DB_NAME,`n    ))`n````n`n### 2. Credential Standardization`nMigrated the system to a purely alphanumeric password (`tesepassword123`) to eliminate all parsing ambiguities across the stack.`n`n### 3. Deployment Script Modernization`nUpdated `tese.ps1` to automatically manage the new granular variables and ensure `.env` generation is consistent with the code changes.`n`n### 4. Linux Compatibility`nUpdated Dockerfiles to include `RUN sed -i 's/\r$//' entrypoint.sh`, ensuring scripts run correctly regardless of the host OS.`n`n## Outcome`nAll backend services are now online and healthy on the VPS (`159.198.42.231`), passing deep health checks and database connectivity tests.
